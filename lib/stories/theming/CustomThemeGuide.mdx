# Creating a Custom Theme

This guide walks you through the process of creating a custom theme for your application using our UI kit.

## Step 1: Understand the Theme Structure

Our theming system is built on CSS custom properties (variables) that define colors, spacing, typography, and other design tokens. These variables are organized into several categories:

- **Base colors**: Primary, secondary, success, warning, danger, info
- **Semantic colors**: Background, foreground, border
- **Typography**: Font sizes, line heights
- **Spacing**: Padding and margin values
- **Border radius**: Corner rounding values
- **Shadows**: Box shadow values
- **Transitions**: Animation durations

## Step 2: Plan Your Theme

Before writing any code, plan your theme by defining your color palette and other design tokens. Consider the following:

- **Brand colors**: What are your primary and secondary brand colors?
- **Semantic colors**: How will you map your brand colors to semantic roles?
- **Dark mode**: How will your colors adapt for dark mode?
- **Typography**: What font sizes and line heights will you use?
- **Spacing**: What spacing scale will you use?
- **Border radius**: How rounded should corners be?

## Step 4: Import Your Theme

Import your theme CSS file in your application. If you're using a bundler like webpack or Vite, you can import it in your main JavaScript/TypeScript file:

```jsx
// Import the default styles
import 'your-ui-kit/dist/style.css';

// Import your custom theme (after the default styles to override them)
import './theme.css';
```

If you're not using a bundler, you can include the CSS files in your HTML:

```html
<link rel="stylesheet" href="path/to/your-ui-kit/dist/style.css">
<link rel="stylesheet" href="path/to/theme.css">
```

## Step 5: Use the ThemeProvider

Wrap your application with the `ThemeProvider` component to enable theme switching:

```jsx
import { ThemeProvider } from 'simple-ui';

function App() {
  return (
    <ThemeProvider>
      {/* Your application content */}
    </ThemeProvider>
  );
}
```

## Step 6: Test Your Theme

Test your theme with various components to ensure it looks good in both light and dark modes. Pay special attention to:

- **Color contrast**: Ensure text is readable against backgrounds
- **Component states**: Check hover, focus, active, and disabled states
- **Responsive design**: Test on different screen sizes
- **Accessibility**: Verify that your theme meets accessibility standards

## Advanced: Theme Switching

If you want to support multiple themes beyond just light and dark, you can create a custom theme switcher:

```jsx
import { useState, useEffect } from 'react';
import { ThemeProvider } from 'simple-ui';

// Define your themes
const themes = {
  default: {
    light: '',
    dark: 'dark',
  },
  blue: {
    light: 'theme-blue',
    dark: 'theme-blue dark',
  },
  green: {
    light: 'theme-green',
    dark: 'theme-green dark',
  },
};

function CustomThemeProvider({ children }) {
  const [theme, setTheme] = useState('default');
  const [mode, setMode] = useState('light');
  
  // Apply the theme and mode classes to the body
  useEffect(() => {
    document.body.className = themes[theme][mode];
  }, [theme, mode]);
  
  return (
    <ThemeProvider defaultTheme={mode}>
      <div className="theme-controls">
        <div>
          <label>Theme:</label>
          <select value={theme} onChange={(e) => setTheme(e.target.value)}>
            <option value="default">Default</option>
            <option value="blue">Blue</option>
            <option value="green">Green</option>
          </select>
        </div>
      </div>
      {children}
    </ThemeProvider>
  );
}
```

## Best Practices

1. **Use a color palette generator**: Tools like [Coolors](https://coolors.co/) or [Adobe Color](https://color.adobe.com/) can help you create harmonious color palettes.

2. **Check contrast ratios**: Use tools like [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) to ensure your text has sufficient contrast against backgrounds.

3. **Create a theme design system**: Document your theme variables and how they should be used to maintain consistency.

4. **Test with real components**: Don't just define variables; test them with actual components to see how they look in practice.

5. **Consider color blindness**: Test your theme with color blindness simulators to ensure it's accessible to all users.

6. **Use semantic variable names**: Name your variables based on their purpose, not their value (e.g., `--color-primary` instead of `--color-blue`).

7. **Maintain dark mode parity**: Ensure all features and components work equally well in both light and dark modes.

## Conclusion

Creating a custom theme allows you to align the UI kit with your brand identity while maintaining the functionality and accessibility of the components. By following this guide, you can create a cohesive and professional-looking theme for your application.
