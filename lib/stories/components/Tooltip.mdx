# Tooltip Component

The Tooltip component provides a way to display additional information when users hover over an element. It's commonly used for providing context, explanations, or additional details about UI elements.

## Features

- Multiple styling variants (default, light, primary, success, warning, danger)
- Customizable positioning (top, right, bottom, left)
- Customizable alignment (start, center, end)
- Optional arrow indicator
- Customizable delay duration
- Accessible with proper ARIA attributes
- Support for rich content

## Component Structure

The Tooltip component system consists of several components that work together:

- `TooltipProvider`: Provider component that should wrap your application or the section where tooltips are used
- `Tooltip`: The main container component
- `TooltipTrigger`: The element that triggers the tooltip
- `TooltipContent`: The content of the tooltip
- `TooltipComponent`: A convenient wrapper component that combines the above components

## Basic Usage

<Canvas of={TooltipStories.Default} />

```tsx
import { TooltipComponent, TooltipProvider } from '@/components';

<TooltipProvider>
  <TooltipComponent content="This is a tooltip">
    <Button>Hover me</Button>
  </TooltipComponent>
</TooltipProvider>
```

## Rich Content

You can include rich content in tooltips by using the lower-level components:

<Canvas of={TooltipStories.WithRichContent} />

```tsx
<Tooltip>
  <TooltipTrigger asChild>
    <Button>Rich Content</Button>
  </TooltipTrigger>
  <TooltipContent className="max-w-[300px] p-4">
    <div className="flex flex-col gap-2">
      <h3 className="text-sm font-semibold">Rich Tooltip Content</h3>
      <p className="text-xs">
        This tooltip contains rich content with multiple elements, including
        text, headings, and even a button.
      </p>
      <Button size="small" variant="outline" className="mt-2">
        Action
      </Button>
    </div>
  </TooltipContent>
</Tooltip>
```

## Disabled Tooltip

You can disable the tooltip using the `disabled` prop:

## Accessibility

The Tooltip component follows accessibility best practices:

- Uses proper ARIA attributes
- Supports keyboard navigation
- Provides appropriate focus management
- Includes appropriate timing delays

## API Reference

### TooltipComponent

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `content` | `React.ReactNode` | - | The content to display in the tooltip |
| `children` | `React.ReactNode` | - | The element that triggers the tooltip |
| `variant` | `'default' \| 'light' \| 'primary' \| 'success' \| 'warning' \| 'danger'` | `'default'` | The visual style of the tooltip |
| `hideArrow` | `boolean` | `false` | Whether to hide the arrow |
| `delayDuration` | `number` | `300` | The delay in milliseconds before showing the tooltip |
| `sideOffset` | `number` | `4` | The offset in pixels from the trigger |
| `side` | `'top' \| 'right' \| 'bottom' \| 'left'` | `'top'` | The side of the trigger to show the tooltip |
| `align` | `'start' \| 'center' \| 'end'` | `'center'` | The alignment of the tooltip relative to the trigger |
| `defaultOpen` | `boolean` | - | Whether the tooltip is open by default |
| `open` | `boolean` | - | Whether the tooltip is open (controlled) |
| `onOpenChange` | `(open: boolean) => void` | - | Callback when the open state changes |
| `className` | `string` | - | Additional class name for the tooltip content |
| `disabled` | `boolean` | `false` | Whether to disable the tooltip |
